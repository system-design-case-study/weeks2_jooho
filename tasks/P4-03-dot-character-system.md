# P4-03: 도트 캐릭터 시스템

## 메타정보
- Phase: 4
- 의존성: P4-02
- 예상 소요: 45분
- 난이도: 중

## 목표
> 16x16 픽셀아트 도트 캐릭터가 Canvas에 렌더링되고, 사용자 ID 기반 HSL 색상이 자동 생성되며, 상태(자기/친구 반경 내/친구 반경 밖/비친구)에 따른 시각적 구분이 적용된다.

## 컨텍스트
> design.md "도트 캐릭터 시스템" 섹션의 전체 명세를 구현한다. 각 사용자가 고유한 색상의 도트 캐릭터로 표시되며, 친구 관계와 검색 반경에 따라 색상/투명도가 변한다. Canvas 렌더링 루프(P4-02)에서 매 프레임 그려진다.

## 상세 요구사항
1. 도트 캐릭터 렌더링 함수
   - 16x16 픽셀아트 스타일 (Canvas에서 사각형 기반 렌더링)
   - 간단한 캐릭터 형태 (원 또는 단순한 픽셀 패턴)
   - Canvas의 좌표계에 맞춰 적절한 크기로 스케일링
2. HSL 색상 자동 생성
   - `hue = hash(userId) % 360` (design.md 명세)
   - 채도(saturation): 고정값 (예: 70%)
   - 명도(lightness): 고정값 (예: 50%)
   - 간단한 해시 함수: userId 문자열의 charCode 합 % 360
3. 상태별 시각적 구분 (design.md)
   - **자기 캐릭터**: 1.5배 크기, 드래그 가능 (드래그는 P4-04에서 구현)
   - **친구 (반경 내)**: 원래 HSL 색상
   - **친구 (반경 밖)**: 회색 (`hsl(0, 0%, 60%)`)
   - **비친구**: 반투명 (`globalAlpha = 0.3`)
4. 이름 라벨 표시
   - 캐릭터 위에 사용자 ID 텍스트 렌더링
   - 폰트 크기: 10~12px
   - 배경 반투명 박스로 가독성 확보
5. 렌더링 루프 통합
   - P4-02의 `requestAnimationFrame` 루프에 캐릭터 렌더링 추가
   - 렌더링 순서: 배경/그리드 → 비친구 (뒤) → 친구 (중간) → 자기 캐릭터 (앞)
   - WebSocket으로 수신한 `useRef` 데이터를 읽어 매 프레임 그림

## 설정 참조
```typescript
// 색상 생성 (design.md)
function generateColor(userId: string): string {
  const hash = userId.split('').reduce((acc, char) => acc + char.charCodeAt(0), 0);
  const hue = hash % 360;
  return `hsl(${hue}, 70%, 50%)`;
}
```

## 테스트 기준
- [ ] 자기 캐릭터가 Canvas에 렌더링되는지 확인
- [ ] 서로 다른 userId에 대해 서로 다른 색상이 생성되는지 확인
- [ ] 같은 userId에 대해 항상 같은 색상이 생성되는지 확인
- [ ] 자기 캐릭터가 다른 캐릭터보다 1.5배 큰지 확인
- [ ] 반경 내 친구가 원래 색상으로 표시되는지 확인
- [ ] 반경 밖 친구가 회색으로 표시되는지 확인
- [ ] 비친구가 반투명으로 표시되는지 확인
- [ ] 이름 라벨이 캐릭터 위에 가독성 있게 표시되는지 확인

## 주의사항
> - design.md: 캐릭터 크기 16x16은 "픽셀아트" 기준. Canvas에서 실제로 표시되는 크기는 좌표 공간 대비 적절히 스케일링 필요
> - design.md: 색상 체계 — 반경 내 초록색이 아니라 사용자별 HSL 색상. "반경 내: 초록색"은 design.md의 핵심 기능 섹션에 언급되나, 도트 캐릭터 시스템 섹션에서는 HSL 원래 색상 유지로 명시. 도트 캐릭터 시스템 섹션을 우선 적용
> - lessons.md: 60 FPS 유지를 위해 매 프레임 전체 Canvas를 다시 그리는 방식. 7~10명 수준에서 성능 문제 없음
