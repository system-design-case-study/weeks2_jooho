# P6-01: Consistent Hash Ring 단위 테스트

## 메타정보
- Phase: 6
- 의존성: P2-01
- 예상 소요: 45분
- 난이도: 중

## 목표
> Consistent Hash Ring의 핵심 동작(균등 분포, 최소 재배치, wrap-around, 결정론적 매핑)을 JUnit 5 단위 테스트로 검증한다.

## 컨텍스트
> Consistent Hash Ring은 사용자 채널(`user:{id}`)을 Redis Pub/Sub 노드에 매핑하는 핵심 모듈이다. Hash Ring의 정확성이 보장되지 않으면 메시지 라우팅 오류로 위치 업데이트가 유실된다. design.md에 명시된 대로 MurmurHash3 128bit (Guava) + TreeMap + ceilingEntry 패턴으로 구현되며, 물리 노드당 150개의 가상 노드를 사용한다. 이 테스트는 구현 정확성을 사전에 검증하여, 이후 Redis Pub/Sub 통합 시 Hash Ring 자체의 문제를 배제할 수 있게 한다.

## 상세 요구사항

### 1. 균등 분포 검증
- 가상 노드 150개 x 물리 노드 2개 구성에서 1000개의 랜덤 키를 분배
- 각 노드에 할당된 키 수의 편차가 20% 이내인지 검증
- 이상적 분포(500:500)에서 최대 허용 범위: 400~600

### 2. 노드 추가/제거 시 최소 재배치 검증
- 2개 노드 → 3개 노드로 확장 시 이동하는 키 수 측정
- 이상적 재배치 비율: 전체 키의 약 1/3 (새 노드가 담당할 비율)
- 실제 이동 키 수가 전체의 50%를 초과하지 않는지 검증
- 반대로, 3개 → 2개로 축소 시에도 동일 원칙 적용 확인

### 3. Wrap-around 테스트
- TreeMap의 `ceilingEntry()`가 `null`을 반환하는 경우 `firstEntry()`로 fallback 하는지 검증
- Hash 공간의 최대값(2^32 - 1)에 가까운 키가 Ring의 첫 번째 노드에 매핑되는지 확인
- 경계값 테스트: hash 값이 0, 2^32-1, 중간값인 경우

### 4. 결정론적 매핑 보장
- 동일 입력(채널 키 + 노드 목록)에 대해 항상 동일 노드를 반환하는지 검증
- 1000번 반복 호출 시 결과 일관성 확인
- 노드 목록의 순서가 달라도 동일 매핑 결과를 보장하는지 검증

### 5. 가상 노드 네이밍 검증
- `"redis-pubsub-1#0"` ~ `"redis-pubsub-1#149"` 형태로 가상 노드가 생성되는지 확인
- 물리 노드 이름이 정확하게 추출되는지 검증

## 테스트 기준
- [ ] 2개 노드, 가상 노드 150개 구성에서 1000개 키 분배 시 편차 20% 이내
- [ ] 노드 추가 시 이동 키 수가 전체의 50% 미만
- [ ] `ceilingEntry()` null → `firstEntry()` fallback 정상 동작
- [ ] 동일 입력 → 동일 노드 매핑 1000회 반복 검증 통과
- [ ] 가상 노드 네이밍 규칙 준수 확인
- [ ] 모든 테스트가 JUnit 5에서 통과
