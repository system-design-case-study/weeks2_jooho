### 권고사항

1. **P2-05 / P3-04 역할 명확화**: P2-05는 FriendService(도메인 서비스 레이어), P3-04는 FriendController(REST API 레이어)로 명확히 분리됨. 수정 완료.

2. **SYSTEM_STATE WebSocket push 로직**: 현재 REST API(P3-05)로만 데이터를 제공하지만, design.md에 `SYSTEM_STATE` WebSocket 메시지 타입이 정의되어 있음. P3-05 구현 시 REST API 외에 주기적 또는 이벤트 기반 WebSocket push 로직도 함께 구현할 것을 권고.

3. **P3-01 크기 주의**: WebSocket 핸들러의 `afterConnectionEstablished()`에서 7단계를 처리(ID 부여, 위치 저장, 친구 로드, 캐시 조회, Pub/Sub subscribe, publish, INIT 전송). 1시간 범위 상한선이므로 구현 시 단계별로 검증하면서 진행할 것.

4. **Phase 6 테스트 실행 시점**: P6-01(단위 테스트)과 P6-02(통합 테스트)는 Phase 2 완료 직후 실행 가능. Phase 6이지만 의존성상 Wave 3~4에서 병렬 착수 가능하므로, 조기에 실행하여 핵심 모듈 정확성을 사전 검증할 것을 권고.

5. **순환 의존성**: 없음 확인 완료. 모든 의존성이 Phase 번호 증가 방향 또는 동일 Phase 내에서 상위 → 하위 방향.
