# P6-05: E2E 시나리오 테스트

## 메타정보
- Phase: 6
- 의존성: P4 전체
- 예상 소요: 45분
- 난이도: 중

## 목표
> 브라우저 탭 2개로 접속하여 친구 추가, 위치 이동, 실시간 반영, 거리 필터링, 전파 경로 로그를 수동으로 검증하는 E2E 시나리오 체크리스트를 작성하고 실행한다.

## 컨텍스트
> 이 프로젝트는 "주변 친구(Nearby Friends)" 분산 시스템의 동작을 XY 좌표 평면 위에서 실시간 시각화하는 학습 프로젝트다. design.md에 따르면 최대 7~10명 동시 접속 규모이며, 별도 관리자 모드 없이 모든 탭이 동일한 뷰를 가진다. 이 E2E 테스트는 자동화 테스트로 커버하기 어려운 시각적 확인과 사용자 경험을 수동으로 검증한다.

## 상세 요구사항

### 1. 접속 및 초기화 시나리오
- 탭 A를 열어 `http://localhost` 접속
- 자동으로 userId 부여 확인 (예: `user-1`)
- XY 좌표 캔버스에 도트 캐릭터가 랜덤 위치에 배치되는지 확인
- 도트 캐릭터 색상이 userId 기반 HSL 색상으로 자동 생성되는지 확인
- 자기 캐릭터가 1.5배 크기로 표시되는지 확인
- 탭 B를 열어 동일 URL 접속 → 별도의 userId 부여 확인 (예: `user-2`)

### 2. 친구 추가 시나리오
- 탭 A에서 탭 B의 userId(`user-2`)를 친구로 추가
- 탭 B에서 탭 A의 userId(`user-1`)를 친구로 추가
- 양쪽 친구 목록에 상대방이 표시되는지 확인
- 상대방의 도트 캐릭터가 캔버스에 표시되는지 확인 (반투명이 아닌 원래 색상)

### 3. 위치 이동 → 실시간 반영 시나리오
- 탭 A에서 도트 캐릭터를 드래그하여 위치 이동
- 탭 B의 캔버스에서 탭 A의 캐릭터 위치가 실시간으로 변경되는지 확인
- 체감 지연이 200ms 이내인지 확인 (design.md 비기능 요구사항)
- 반대로 탭 B에서 위치 이동 → 탭 A에 반영 확인

### 4. 반경 내/외 필터링 동작 시나리오
- 탭 A의 캐릭터를 (100, 100) 근처로 이동
- 탭 B의 캐릭터를 (200, 200) 근처로 이동 → 거리 약 141 → 기본 반경 200 이내
- 탭 A에서 탭 B의 캐릭터가 초록색(반경 내)으로 표시되는지 확인
- 탭 B의 캐릭터를 (900, 900)으로 이동 → 거리 약 1131 → 반경 밖
- 탭 A에서 탭 B의 캐릭터가 회색(반경 밖)으로 변경되는지 확인
- 검색 반경 슬라이더(Phase 2) 조절 시 필터링 기준이 실시간 변경되는지 확인

### 5. 전파 경로 로그 확인 시나리오
- 탭 A에서 위치 이동 후 전파 경로 로그 패널 확인 (Phase 2)
- 로그에 다음 단계가 표시되는지 확인:
  - `user-1 → ws-X (receive)`
  - `→ redis-pubsub-X (publish)`
  - `→ redis-pubsub-X (subscribe_receive)`
  - `→ ws-Y (deliver) → user-2`
- 경유 노드 ID가 실제 인프라와 일치하는지 확인

### 6. 비정상 시나리오
- 탭 A를 닫은 후 탭 B에서 탭 A의 캐릭터가 사라지거나 비활성 표시되는지 확인
- 탭 A를 다시 열면 새로운 userId가 부여되는지 확인 (세션 비유지)
- 네트워크 끊김 시뮬레이션 (브라우저 개발자 도구 Offline 모드) → 재연결 동작 확인

## 테스트 기준
- [ ] 탭 2개 접속 시 각각 고유 userId 부여
- [ ] 도트 캐릭터가 userId 기반 고유 색상으로 표시
- [ ] 친구 추가 후 상대방 캐릭터가 캔버스에 표시
- [ ] 한 쪽 위치 이동 → 다른 쪽 캔버스에 200ms 이내 반영
- [ ] 반경 내 친구: 초록색 표시, 반경 밖 친구: 회색 표시
- [ ] 전파 경로 로그에 4단계 경유 정보 표시 (Phase 2)
- [ ] 탭 닫기 → 상대방에게 반영
- [ ] 시나리오 수동 확인 체크리스트 전체 통과

## 주의사항
> - design.md: 모든 탭이 동일한 뷰를 가짐 (자기 위치 맵 + 시스템 상태 패널). 별도 관리자 모드가 없으므로 동일 URL에서 테스트
> - domain-insights.md 경고: WebSocket 서버는 Stateful이므로 컨테이너 재시작 시 모든 연결 손실. 테스트 중 서버 재시작이 필요하면 모든 탭을 새로고침해야 함
> - lessons.md 경고: Canvas Retina 미대응 시 도트 캐릭터가 흐릿하게 보임. `devicePixelRatio` 처리가 적용되었는지 시각적으로 확인
> - design.md: 비친구는 반투명 처리. 친구 추가 전과 후의 캐릭터 표시 차이를 확인할 것
> - 이 테스트는 수동 시나리오이므로 결과를 체크리스트 형태로 기록하여 추적 가능하게 관리할 것
